     GundariuM — Gunpla NFT TCG on Base

     Plan

     Context

     Building a blockchain-based NFT Trading Card Game on Coinbase's Base
     network, starting from the existing basement-gunpla-next scaffold
     (Next.js 16.1.6 / React 19 / Tailwind v4 / TypeScript). Users
     photograph their real Gundam model kits (Gunpla), mint them as unique
     NFT trading cards with AI-generated lore-accurate stats, then use
     those cards in a turn-based RPG battle system (PVE campaign + PVP
     faction arena). Economy runs on $GNDM token (upgrades/battles/prizes)
     and USDC (minting + cosmetics).

     ---
     Tech Stack to Add

     npm packages (production)

     @coinbase/onchainkit   wagmi   viem   @tanstack/react-query
     @anthropic-ai/sdk      pinata-web3   zustand
     @radix-ui/react-dialog @radix-ui/react-toast @radix-ui/react-tabs
     framer-motion          react-dropzone   clsx   tailwind-merge

     Contracts (Foundry, in /contracts subfolder)

     forge install OpenZeppelin/openzeppelin-contracts-upgradeable
     forge install OpenZeppelin/openzeppelin-foundry-upgrades

     ---
     Smart Contracts (/contracts/src/)

     1. GunplaCard.sol — ERC-721 UUPS Upgradeable NFT

     Inherits: ERC721Upgradeable, ERC721URIStorageUpgradeable,
     ERC721EnumerableUpgradeable, OwnableUpgradeable, UUPSUpgradeable

     On-chain CardTraits struct:
     - Identity: name, series, faction, pilotName
     - Stats: rarity (uint8 0–4), hp (uint16), armorType (uint8)
     - Weapons: primaryWeapon/Damage, secondaryWeapon/Damage,
     tertiaryWeapon/Damage, specialAttack/Damage
     - Cosmetics (mutable): repaintColor, decalId

     Key functions: initialize(), mintCard(to, tokenUri, traits),
     updateCosmetics(tokenId, color, decalId), getTraits(tokenId)

     Minting: accepts USDC (transferFrom → contract). Cosmetics: separate
     USDC fee, callable only by token owner.

     2. GundaniumGame.sol — Battle Logic (Hybrid on/off-chain)

     BattleSession struct: players, card IDs, GNDM stake, type (PVE/PVP),
     status, arcId, winner.

     Key functions: startPVE(cardId, arcId), startPVP(cardId, stakeAmount),
      joinPVP(sessionId, cardId), settleBattle(sessionId, winner, 
     EIP712sig), claimPVEReward(sessionId), setTrustedResolver(address)

     3. PrizePool.sol — Daily/Weekly/Monthly Escrow

     depositGndm(), depositUsdc(), distribute(poolType, winners[], 
     gndmShares[], usdcShares[], adminSig)

     ---
     Key Architecture Decisions

     Battle Resolution: Hybrid (Off-chain engine + On-chain settlement via 
     EIP-712)

     - Why not fully on-chain: Each turn costs gas + 2s block time →
     unplayable
     - Why not fully off-chain: Centralized, unverifiable, defeats
     blockchain purpose
     - Hybrid approach:
       a. startPVP/startPVE locks stakes on-chain
       b. All move resolution runs on the game server (/api/battle/resolve)
       c. Server signs BattleResult{sessionId, winner, finalHp, timestamp}
     with EIP-712
       d. Client submits signed result → settleBattle() verifies sig,
     distributes GNDM
       e. RNG seed = keccak256(playerCommitment_A, playerCommitment_B)
     (commit-reveal, unmanipulable)

     AI Trait Generation: Claude claude-opus-4-6 Vision (server-side only)

     Photo → base64 → /api/analyze-gunpla → Claude vision → structured JSON
      traits → user reviews/edits → /api/mint-metadata → Pinata IPFS →
     on-chain mint

     Rarity Visual Treatment (Tailwind v4 CSS vars in globals.css)

     - Common: silver border | Uncommon: green holo | Rare: blue metallic |
      Ultra Rare: purple gradient | Legendary: animated gold beam
